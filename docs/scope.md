# Scope

OpsFlow Lab is a public frontend portfolio and engineering case study.

## Product Identity

A hyperfast, motion-rich frontend systems lab inspired by McMaster-Carr-style speed and React Bits-level visual polish.

## Primary Goals

- Demonstrate frontend architecture discipline.
- Build a visually impressive portfolio shell.
- Practice aggressive caching and prefetching.
- Show dense, fast, keyboard-friendly navigation.
- Build business-systems-oriented interfaces.
- Showcase motion without sacrificing performance.
- Maintain a clean public Git history.
- Document decisions and implementation tradeoffs.

## Locked v1 Modules

1. Landing / Portfolio Shell
2. Hyper Catalog
3. Process Simulator
4. Workflow Builder
5. Analytics Dashboard
6. Design System Playground
7. Case Studies

## MVP Scope

The MVP proves the project concept publicly.

Included:

- Landing page
- Lab shell
- React Bits visual system
- Command palette
- Intent prefetching
- Hyper Catalog
- Dexie cache
- README
- Architecture docs
- One strong case study

## Portfolio v1 Scope

Portfolio v1 turns the MVP into a serious frontend portfolio.

Included:

- MVP
- Process Simulator
- Workflow Builder
- Analytics Dashboard
- Design System Playground
- Multiple case studies
- Performance writeups
- Accessibility notes
- Clean public repo history

## Portfolio v2 Stretch Scope

Portfolio v2 is allowed only after Portfolio v1 is stable.

Possible additions:

- Service worker
- Cache visualization lab
- Storybook polish
- Playwright flows
- Advanced dashboard polish
- Redis demo only through ADR
- Supabase/tRPC only through ADR

## Explicitly Out of Scope for v1

- Supabase auth
- real users
- real database persistence
- Redis cache
- tRPC API layer
- AI backend
- payments
- CMS
- admin panel
- realtime collaboration
- multi-user accounts

## Scope Control Rule

If a feature does not strengthen frontend portfolio value, speed, interaction quality, or documentation discipline, it does not enter v1.
